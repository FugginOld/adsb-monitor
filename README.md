# ADS-B Stack Monitor

A self-hosted web dashboard for monitoring an ADS-B receiver station. Shows live feeder health, signal quality, uptime history, version status, and lets you configure and restart services — all from one page.

---

## What it does

| Area | Details |
|---|---|
| **Live status** | Active / down badge for every feeder and service, with last-seen time |
| **Uptime history** | 7-day daily bars + aggregate % per service, backed by SQLite |
| **Dual-band** | 1090 MHz ADS-B + 978 MHz UAT on two RTL-SDRs, merged onto one map |
| **Per-band signal** | 1090 block (RSSI / SNR / noise) and 978 block (RSSI), shown only for present bands; Airspy block hidden when absent |
| **readsb deep stats** | Aircraft count, message rate, range, signal metrics |
| **Version checking** | Current vs latest for each stack component |
| **Settings** | Per-SDR gain + bias-tee (1090 / 978), sharing keys, lat/lon/altitude, receiver options in-browser |
| **Log streaming** | Live tail of any feeder's systemd journal or Docker log (with Copy), plus a whole-system journal window in Settings |
| **Service control** | Restart any service (admin port only) |
| **Backup / restore** | Download config or graph data (collectd RRDs); import either back |

## Architecture

```
app.py  (single Python file, Flask)
│
├── Host seam — LinuxHost / FakeHost
│   run(cmd) · read_text · read_json · write_text
│   Never raises; failures degrade to Result{ok=False}/None.
│
├── Init adapter — SystemdAdapter / OpenRCAdapter / NullAdapter
│   status(svc) · action(svc, act) · running_since(svc)
│   Detected once at startup (detect_init); swappable in tests.
│
├── Config store — one adapter per format
│   ini_flat · shell_vars · piaware · docker
│
├── Uptime ledger — SQLite service_events table
│   fold_uptime() pure function, unit-testable without DB
│
├── Feeder probe — FeederHealth{status, detail, last_seen, running_for}
│   Single kind-dispatch; callers never branch on service vs docker.
│
└── Flask routes  (admin :5000 · read-only :5001)
    /api/status · /api/alerts · /api/stats/* (incl. /api/stats/bands)
    /api/settings/* (incl. /api/settings/sdr/<1090|978>)
    /api/service/<name>/restart · /api/logs/<key>
    /api/backup · /api/backup/graphs · /api/restore · /api/restore/graphs
```

### Two ports

| Port | Purpose |
|---|---|
| `5000` (admin) | Full access — settings writes, service restarts, backup |
| `5001` (read-only) | Status + stats only — safe to expose via reverse proxy |

Configurable via `ADMIN_PORT` / `READONLY_PORT` environment variables.

### Feeder kinds

`feeders.ini` declares each monitored unit as either:

- `[service:<unit>]` — a systemd unit or OpenRC service
- `[docker:<container>]` — a Docker container

### Init system support

At startup `detect_init()` probes the host and selects the right adapter:

| Init system | Adapter | Detection |
|---|---|---|
| systemd | `SystemdAdapter` | `systemctl --version` succeeds |
| OpenRC (Alpine / minimal SBC) | `OpenRCAdapter` | `rc-service --version` succeeds |
| Unknown | `NullAdapter` | service control endpoints return a clear error |

### Supported SDRs

| SDR | Decoder |
|---|---|
| Airspy Mini | airspy_adsb, 6 MSPS |
| Airspy R2 | airspy_adsb, 12 MSPS |
| RTL-SDR Blog V4 / generic RTL2832U | readsb |
| FlightAware Pro Stick / Plus | readsb |
| Nooelec NESDR SMArt / SMArTee | readsb |
| SDRplay RSP1A / RSPdx | SDRplay API + dump1090 (partly manual) |
| **2× RTL-SDR (dual-band)** | readsb (1090) + dump978-fa (978 UAT), merged on one map |

## Files

```
app.py              Flask application (single file)
feeders.ini         Feeder declarations (auto-generated by installer)
history.db          SQLite uptime + metrics history (auto-created)
static/index.html   Single-page frontend
installer/
  install-stack.sh  Full TUI installer
  update-stack.sh   Update wiedehopf components + redeploy monitor
  uninstall.sh      TUI removal checklist
  lib.sh            Shared helpers (pkg mgr, init detection, TUI bootstrap)
  detect-sdr.sh     USB-ID SDR detection
  generate-feeders-ini.sh  Writes feeders.ini from installed services
```

## Supported platforms

| Distro | Status |
|---|---|
| Raspberry Pi OS, Armbian, Debian, Ubuntu | Supported |
| Fedora, CentOS, RHEL, Rocky, Alma | Experimental |
| Arch, Manjaro | Experimental |
| Alpine | Experimental |
| Debian-like (`ID_LIKE=debian`) | Supported |

Set `ADSB_FORCE=1` to skip the compatibility prompt on experimental/unsupported distros.

## Requirements

- Python 3.9+
- `flask` (`pip install flask`)
- `sqlite3` (stdlib)
- systemd **or** OpenRC (service control); monitor-only mode works without either
- **Dual-band (978 UAT) only:** `dump978-fa` + `skyaware978` (built by the installer),
  `soapysdr-module-rtlsdr`, and `rtl-sdr` for `rtl_test` / `rtl_eeprom` / `rtl_biast`.
  See [CREDIT.md](CREDIT.md) for the full list.
